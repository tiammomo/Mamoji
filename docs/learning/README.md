# Mamoji 使用教学

这份文档用于教学本项目如何启动、登录、理解业务模块，并完成一次完整的企业经营管理体验。建议配合 README 先完成本地启动，再按照下面的课程顺序演示。

## 学习目标

完成本教程后，你应该能做到：

- 理解 Mamoji 的定位：面向初创公司和小团队的企业经营记账与人员经营管理系统。
- 独立启动前后端，并确认本地 SQLite 演示数据可用。
- 使用管理员账号完成首页、经营流水、预算、报表、资金账户、税务、HR 和系统设置的基础操作。
- 理解公司主体、家庭主体、多地区政策、角色权限和权限矩阵的设计思路。
- 知道后续开发应当优先扩展哪些后端模型和前端页面。

## 课前准备

确认本机已经安装：

- JDK 21
- Maven 3.9+
- Node.js 20+ 或 24+
- npm 10+

从项目根目录启动后端：

```bash
mvn -f backend/pom.xml spring-boot:run
```

另开一个终端启动前端：

```bash
cd frontend
npm install
npm run dev
```

访问：

- 前端：`http://localhost:33000`
- 后端健康检查：`http://localhost:38080/actuator/health`

## 课程 1：认识系统结构

Mamoji 采用前后端分离结构：

- `frontend/`：Next.js 前端，负责页面、交互、主题、中英文切换和 API 调用。
- `backend/`：Spring Boot 后端，负责认证、经营流水、预算、报表、企业主体、员工、税务等 API。
- `backend/data/mamoji.db`：本地 SQLite 数据库文件，用于演示和开发，不提交到 Git。
- `docs/`：产品定位、架构、权限矩阵和教学文档。

建议先打开下面几份文档建立全局认识：

- 产品定位：`docs/ENTERPRISE_PRODUCT_POSITIONING.md`
- 模块架构：`docs/ENTERPRISE_MODULE_ARCHITECTURE.md`
- 权限矩阵：`docs/ENTERPRISE_PERMISSION_MATRIX.md`
- 多公司与地区政策：`docs/ENTERPRISE_MULTI_COMPANY_POLICY.md`

## 课程 2：首页与主体切换

进入 `http://localhost:33000/dashboard`。

重点观察：

- 本月收入、本月成本、本月利润、预算使用率。
- 当前主体信息，包括主体类型、地区和税务画像。
- 员工数量、人力成本、待处理税费等企业经营指标。

教学建议：

1. 先说明系统不是个人记账工具，而是企业经营工作台。
2. 打开顶部主体切换，展示公司主体与家庭主体。
3. 解释为什么需要家庭主体：创始团队可能存在家庭资金垫付、借款、报销、还款等公司与家庭之间的资金往来。
4. 强调主体切换会影响当前页面的数据范围。

## 课程 3：经营流水

进入 `http://localhost:33000/transactions`。

这个页面用于管理经营收入、成本支出、退款和大额流水。

推荐演示顺序：

1. 使用关键词搜索备注、分类或账户。
2. 使用类型、分类、账户筛选流水。
3. 使用起始日期、结束日期筛选经营周期，日期格式为 `yyyy-MM-dd`。
4. 使用最小金额、最大金额查找特定金额区间。
5. 观察大额流水标签，解释它用于经营风险识别。
6. 点击“录入流水”按钮，通过弹窗新增一笔收入或成本支出。

讲解重点：

- 经营流水是企业财务与经营分析的数据底座。
- 流水需要关联账户、分类和日期，后续预算、报表、税务估算都会依赖这些字段。
- 大额流水、重复流水和异常支出后续可以发展成风控能力。

## 课程 4：经营预算

进入 `http://localhost:33000/budgets`。

经营预算用于控制成本、项目投入和部门费用。

推荐演示顺序：

1. 查看预算总览，理解预算金额、已使用、剩余金额和使用率。
2. 使用起始日期、结束日期筛选预算周期。
3. 按预算状态筛选健康、预警、超支预算。
4. 新增一个预算，设置预算名称、金额、开始日期、结束日期和预警阈值。
5. 回到经营流水新增一笔相关支出，再回来看预算使用率变化。

讲解重点：

- 预算必须有起止时间，否则无法判断费用是否落在预算周期内。
- 企业预算可以按公司、部门、项目、成本类型逐步扩展。
- 当前系统先支持经营预算台账，后续可以扩展审批、预算占用和预算调整记录。

## 课程 5：经营报表

进入 `http://localhost:33000/reports`。

经营报表用于把流水和预算转化为管理层可以理解的指标。

推荐演示顺序：

1. 看经营概览：收入、成本、利润、税费、预算风险。
2. 看趋势：近几个月收入与成本变化。
3. 看分类占比：哪些成本类型占比最高。
4. 看年度视图：每月收入、成本和结余。
5. 看洞察与异常：大额交易、分类环比增加、预算风险。

讲解重点：

- 报表不是简单展示流水，而是帮助管理层回答“公司现在是否健康”。
- 对初创团队来说，应重点关注 burn rate、runway、现金余额、人力成本和税费压力。
- 后续可以增加客户收入集中度、项目毛利、部门成本、应收应付账龄等经营指标。

## 课程 6：资金账户与票据凭证

进入：

- `http://localhost:33000/accounts`
- `http://localhost:33000/receipts`

资金账户用于管理公司现金、银行账户、信用卡等账户余额。

票据凭证用于承载发票、收据、合同付款截图等凭据入口。

教学建议：

1. 在资金账户中查看不同账户的余额和资产负债影响。
2. 说明经营流水必须落到某个账户，账户余额才可追踪。
3. 在票据凭证中演示上传入口，解释后续可扩展 OCR、发票验真和凭证归档。

## 课程 7：税务管理

进入 `http://localhost:33000/tax`。

税务管理用于展示税费事项、申报状态和待缴金额。

推荐演示顺序：

1. 查看待处理税费列表。
2. 观察税种、所属期间、截止日期和状态。
3. 解释地区政策差异：不同公司主体所在地区可能有不同税务规则、申报口径和优惠政策。
4. 结合主体切换说明，多公司多地区时需要按主体管理政策画像。

讲解重点：

- 税务不是孤立模块，它依赖收入、成本、工资、人力成本和主体地区。
- 当前系统展示税费事项台账，后续可以扩展申报日历、税额测算、政策规则库和风险提醒。

## 课程 8：HR 管理

进入：

- `http://localhost:33000/admin/users`
- `http://localhost:33000/admin/compensation`

人员信息页用于管理员工基本信息、部门、岗位、状态和企业角色。

人员薪酬页用于承载工资、社保、公积金、个税估算和月人力成本。

推荐演示顺序：

1. 查看员工列表，按状态筛选在职、试用期、待入职、已离职人员。
2. 查看员工部门、岗位、企业角色和数据范围。
3. 解释员工入职、离职、状态变化会影响权限与人力成本。
4. 进入人员薪酬，讲解工资、社保、公积金、个税估算和月人力成本之间的关系。

讲解重点：

- HR 是企业管理的重模块，不只是账号管理。
- 后续需要扩展劳动合同、考勤、社保公积金缴纳记录、薪资发放、离职交接、权限回收。
- 员工权限应与角色、部门、主体、数据范围绑定。

## 课程 9：权限矩阵

建议结合 `docs/ENTERPRISE_PERMISSION_MATRIX.md` 讲解。

当前系统中的典型角色：

- 创始人/CEO：全公司管理权限。
- 财务管理员：经营流水、账户、预算、报表、税务相关权限。
- HR 管理员：人员信息、入职离职、薪酬和权限相关权限。
- 部门负责人：本部门人员和预算视图。
- 普通员工：本人信息和有限查看能力。
- 只读观察者：只读数据，不允许修改。

教学重点：

- 权限不只区分“能不能进入页面”，还要区分数据范围。
- 数据范围可以是全公司、本部门、本人、只读。
- 离职员工应自动降权或禁用关键权限。

## 课程 10：系统设置与备份

进入：

- `http://localhost:33000/settings`
- `http://localhost:33000/backup`

推荐演示顺序：

1. 在设置中切换语言和主题。
2. 在备份页面查看当前数据状态。
3. 演示导出备份或校验备份文件。

讲解重点：

- 备份能力对本地 SQLite 演示和小团队部署很重要。
- 后续生产化时，需要考虑自动备份、权限审计、操作日志和敏感数据脱敏。

## 练习任务

可以用下面的任务检验学习效果：

1. 新增一笔客户回款，并在经营报表中观察本月收入变化。
2. 新增一笔大额成本支出，并在经营流水中确认大额标签。
3. 新增一个本月预算，再新增一笔该分类支出，观察预算使用率变化。
4. 切换公司主体和家庭主体，理解主体间资金往来为什么需要单独记录。
5. 修改一个员工状态，思考它应该如何影响权限和人力成本。
6. 查看税费事项，判断哪些税费即将到期。

## 开发者学习路径

如果你希望继续开发项目，建议按下面顺序读代码：

1. `backend/src/main/java/com/mamoji/controller`：先看 API 入口。
2. `backend/src/main/java/com/mamoji/service`：理解业务逻辑。
3. `backend/src/main/java/com/mamoji/repository`：理解 SQLite 和内存 Store。
4. `frontend/src/lib/api`：理解前端如何调用后端。
5. `frontend/src/app/(main)`：理解页面结构。
6. `frontend/src/components`：理解复用组件。

优先扩展方向：

- HR：员工详情、劳动合同、社保公积金、考勤、薪酬发放。
- 财务：票据凭证、应收应付、费用报销、付款审批。
- 税务：税额测算、申报日历、政策规则、风险提醒。
- 主体：多公司、多地区政策、家庭主体、主体间转账和借款。
- 审计：操作日志、权限审计、数据导出脱敏。

## 常见问题

### 前端打不开

确认前端服务运行在 `http://localhost:33000`，并且执行过 `npm install`。

### 登录后接口报错

确认后端服务运行在 `http://localhost:38080`，并且前端 API 地址指向 `http://localhost:38080/api/v1`。

### 想重置演示数据

停止后端后删除本地 SQLite 文件，再重新启动后端：

```bash
rm backend/data/mamoji.db
mvn -f backend/pom.xml spring-boot:run
```

注意：这会删除本地演示数据和你刚录入的内容。

### 真实公司数据是否可以提交

不可以。真实公司名称、统一社会信用代码、员工信息、工资、税费、流水和 SQLite 数据库文件都不应提交到 Git。
